Introduction Last updated: 08-04-2020

Welcome to the official documentation for BookAProf REST API! The BookAProf REST API aims to give developers access to data regarding consultation bookings at Singapore Management University (SMU). We hope you find this documentation helpful and encourage you to use our API.

Version Control

Version No. Description Release Date
1.0 First release of the BookAProf REST API 15th March 2020

Authorization

In order to use the BookAProf REST API, you must be a student or faculty member of SMU.

Faculty members are only allowed to use methods specified under the faculty section while students are only allowed to use methods specified under the student section. Users will not be able to use the API without successful authentication. The authentication token that is used for authorization is in the format of email:password, but we will use <token> to represent the authentication token in this documentation.

Making Requests

All queries to BookAProf REST API must be served over HTTPS and need to be presented in this form:

https://bookaprof.herokuapp.com/api/TOKEN/METHOD_NAME

For example:

https://bookaprof.herokuapp.com/api/username@smu.edu.sg:12345/getSlot

We support GET, POST and PUT HTTP methods. Parameters are passed in BookAProf API requests using the URL query string. Users can access the BookAProf API in several ways, including Python scripts and the Postman application. All methods in the BookAProf REST API are case sensitive. The response contains a JSON object with two items – “response” and “status”. They provide the results of the query and the status code respectively.

Faculty Methods

All methods listed in this section are only intended to be used by faculty members of SMU. Students should not try to impersonate a faculty member. Methods in this section generally allow faculty members to create, retrieve, update and delete consultation slots.

createSlot

Allows faculty members to add available consultation slots.

URL https://bookaprof_api.herokuapp.com/api/token/createSlot
Method POST
Returns The details of the newly created slot and status code in JSON format.
Note Consultation slots must be created at least 2 days in advance. For example, if today is Monday, the earliest possible slot that can be created is on Thursday. This is to give students time to book and prepare for the consultation.

The table below highlights the request parameters for the createSlot api.

Field Required Type Description
date Yes String The date of the available slot. Must be in the ‘dd/mm/yyyy’ format, e.g. ‘31/01/2020’.
start_time Yes String The start time of the available slot. Must be in the ‘hh:mm’ 24-hour format, e.g. '14:30’. startTime value must be between ‘08:30’ and ‘18:00’ and minute value has to be either 00 or 30.
end_time Yes String The end time of the available slot. Must be in the ‘hh:mm’ 24-hour format, e.g. '15:30’. endTime value must be between ‘09:00’ and ‘18:30’ and minute value has to be either 00 or 30. Method differs according to consultation type: For ‘private’ consultations: if the total duration is longer than 30 minutes, the consultation slot will be split into multiple slots with 30 minutes duration for each slot. For ‘open’ consultations: the consultation slot will not be split into multiple slots.
type No String The type of consultation. Values can only be either ‘open’ or ‘private’. If unspecified, the default is ‘private’. The difference between ‘open’ and ‘private’ is as follows: ‘private’: only one student is allowed to book the slot. Slot will be unavailable when one student confirms a booking. ‘open’: allows multiple students to book the same slot. Slot will be unavailable when the maximum number of sign-ups is reached.
slots No Integer The maximum number of student bookings allowed for open consultation slots. If unspecified, the default is 1. There is no need to specify the number of slots for private consultations, as it must be 1.
fDesc No String The description written by the faculty member to let students know what topics will be covered or what to prepare. E.g. ‘Python refresher class. Bring bubble tea!’ If unspecified, the default is an empty string.
location No String The location of the available slot. If unspecified, the default is the faculty member’s office location, e.g. ‘SIS RM 4001’.

getSlot

Allows faculty members to get available and/or confirmed consultation slots.

URL https://bookaprof_api.herokuapp.com/api/token/getSlot
Method GET
Returns The details of the available and/or confirmed slots and status code in JSON format.
Note Calling this method without any parameters will return all of the faculty member’s available and/or confirmed slots.
Field Required Type Description
slotID No Integer The ID of the confirmed/available slot, e.g. 1. If this parameter is used, the server will disregard all other parameters and return only the specified slot.
isAvail No String Value can only be either ‘available’ or ‘confirmed’. For ‘private’ consultations: ‘isAvail' shows ‘available’ when no student has booked the slot; when one student books the slot, ‘isAvail’ will be changed to ‘confirmed’ by the system. For ‘open’ consultations: ‘isAvail' will show ‘available’ until the maximum number of students for that slot is reached. When the maximum capacity for the slot is reached, ‘isAvail' will be changed to ‘confirmed’ by the system.
end_time Yes String The end time of the available slot. Must be in the ‘hh:mm’ 24-hour format, e.g. '15:30’. endTime value must be between ‘09:00’ and ‘18:30’ and minute value has to be either 00 or 30. Method differs according to consultation type: For ‘private’ consultations: if the total duration is longer than 30 minutes, the consultation slot will be split into multiple slots with 30 minutes duration for each slot. For ‘open’ consultations: the consultation slot will not be split into multiple slots.
type No String The type of consultation. Values can only be either ‘open’ or ‘private’.
date No String The date of the available and/or confirmed slots. Must be in the ‘dd/mm/yyyy’ format, e.g. ‘31/01/2020’.

bookSlot

Allows students to book an available consultation slot.

URL https://bookaprof_api.herokuapp.com/api/token/bookSlot
Method POST
Returns The details of the slot that the student booked and status code in JSON format.
Note Students must book consultation slots at least 2 days in advance. E.g. if today is Monday, the earliest possible slot that a student can book is on Thursday.
Field Required Type Description
slotID Yes Integer The ID of the available slot, e.g. 505.
sDesc No String Any information that the student would like to inform the faculty member of. This may include subject area(s) that the student wants to focus on, special requests to the faculty member, group members’ details etc. E.g. “Team 5 project consultation on database design”. If unspecified, the default is an empty string.

Status Codes & Error Handling

Status codes indicate to users whether their API requests have been made successfully or not. This section aims to list out the various possible status codes that will be returned after an API request.

Depending on the status code returned, users can better know how to handle possible errors that may arise when making the API request.

Successful

The table below lists all the status codes that indicate that your API request was successful. If you see any of these status codes: CONGRATULATIONS! The sample responses shown in the Available Methods section above show possible examples of responses from a successful API call.

Status Code Method Notes Response body
200 [OK} GET Used for GET methods, e.g. getting all available slots The requested data as JSON
201 [Created] POST Used for POST methods, which will result in creating new data, e.g. booking a slot The created data as JSON

Unsuccessful

The table below lists all the status codes that indicate that your API request was unsuccessful. Our group is unable to identify all errors that may occur in your API request. Hence, the list of status codes for errors listed below is non-exhaustive and you may encounter an error status code that is not listed below. If you see any of these status codes, it may be necessary to check the values that you input for each parameter or the url of your API request.
