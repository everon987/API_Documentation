Authorization

In order to use the BookAProf REST API, you must be a student or faculty member of SMU.

Faculty members are only allowed to use methods specified under the faculty section while students are only allowed to use methods specified under the student section. Users will not be able to use the API without successful authentication. The authentication token that is used for authorization is in the format of email:password, but we will use <token> to represent the authentication token in this documentation.

Making Requests

All queries to BookAProf REST API must be served over HTTPS and need to be presented in this form:

https://bookaprof.herokuapp.com/api/TOKEN/METHOD_NAME

For example:

https://bookaprof.herokuapp.com/api/username@smu.edu.sg:12345/getSlot

We support GET, POST and PUT HTTP methods. Parameters are passed in BookAProf API requests using the URL query string. Users can access the BookAProf API in several ways, including Python scripts and the Postman application. All methods in the BookAProf REST API are case sensitive. The response contains a JSON object with two items – “response” and “status”. They provide the results of the query and the status code respectively.

Availability

This section summarizes the methods that can be used by faculty members to add, view, edit or delete available slots on BookAProf!.

Add Available Slots
Field Type Description
date String The date of the available slot. Must be in the ‘dd/mm/yyyy’ format, e.g. ‘31/01/2020’.
start_time String The start time of the available slot. Must be in the ‘hh:mm’ 24-hour format and only with mm=‘00’ or mm=‘30’, e.g. '14:30’.
end_time String The end time of the available slot. Must be in the ‘hh:mm’ 24-hour forma and only with mm=‘00’ or mm=‘30’t, e.g. '15:30’. Method differs according to consultation type: ’private’: if the specified end time is longer than 30 minutes, the consultation slot will be split into multiple slots with 30 minutes duration. ‘open’: consultation slot will not be split into multiple slots. More details of ‘type’ can be found here.
description* String The description of the available slot. If unspecified, the default is ‘-’. For students’ to understand what to prepare for the consultation, e.g. ‘Please bring along your own bubble tea for extra consultation boost’.
location* String The location of the available slot. If unspecified, the default is the faculty member’s office location, e.g. ‘SIS RM 4001’.
type* String The type of consultation. Values can only be either ‘open’ or ‘private’. If unspecified, the default is ‘private’. The difference between ‘open’ and ‘private’ is specified below. ‘private’: for private consultations for groups or individual students; allows only one student to book the slot, slot will be unavailable when one student confirms a booking. ‘open’: for open consultations; allows multiple students to book the same slot, slot will be unavailable when the maximum number of sign ups is reached.
module* String The module name. If unspecified, the default is ‘any module’. Often associated with open consultations, e.g. ‘SMT203’.
slots* Integer The maximum number of bookings allowed. If unspecified, the default is 1. Often associated with open consultations, e.g. 20.
